# Markdown 機能ルール

Markdown の記法を使用する場合は, 以下のルールに従ってください.

> 本ドキュメントでは, 禁止記法の説明のためにコードブロック内でそれらが現れることがありますが, 例示目的であり本文の記法としては使用しません.

## 使用できる記法

以下の記法のみ使用を許可します.

- 見出し (`#` ~ `######`)
- 数値なしの箇条書き
- リンク (`[表示テキスト](https://...)`)
- 打ち消し線
- インラインコード / コードブロック
- 引用
- 数式

太字・斜体・数値付きリストなど, 上記にない記法は一切使用してはいけません.

## 見出し

見出しは, そのあとに続く内容を簡潔に表現するためだけに使用してください.

- 階層構造をもたせるためだけに使用しないこと.
- 意味のない長い表題を使用しないこと.
- 先頭に `1.` や `1.1` のような番号を付けないこと.

正しい使用例:

```markdown
## VS Code を使用するメリットについて

VS Code というエディターについて紹介します.

### 充実した基本機能 (置換・検索)

標準で導入されている検索・置換が強力で, 正規表現やファイル横断検索, 除外設定などを直感的に扱うことができます.
大規模プロジェクトでも目的の箇所を素早く見つけることができ, 変更も一括で安全に反映しやすいため, 作業効率と品質が上がります.

### Git との連携

VS Code は Git との連携機能が標準で備わっており, エディター上でコミットやブランチの切り替え, 差分の確認などを直感的に行うことができます. ターミナルでコマンドを打つことなく視覚的に変更内容を把握できるため, 特に Git 操作に不慣れな方にとっても扱いやすい環境となっています. また, GitHub との連携拡張機能を導入することで, プルリクエストの作成やレビューもエディター内で完結させることができます.
```

## 箇条書き

箇条書きは, 前後の文脈とあわせて内容を見やすく整理するために使用してください.
内容を列挙するためだけに使用してはいけません.

正しい使用例 1:

```markdown
アルゴリズムについて詳しくなることで,

- 問題解決力が上がる: 複雑な課題を分解して, 難しい問題を解くことができるようになる.
- 論理的思考力の向上: 内部で行われる処理を予測し, 直感に頼らない一貫性のある判断や説明ができるようになる.
- コストとリソースの最適化: 限られた資源のなかで, 高速に動くプログラムを実装できるようになる.

といったメリットがあります.
```

正しい使用例 2:

```markdown
パスワードポリシーとして, 以下の条件を満たす必要があります.

- 長さ: 12 文字以上であること.
- 文字種: 英大文字, 英小文字, 数字, 記号をそれぞれ 1 つ以上含むこと.
- 禁止事項: ユーザー名や誕生日など推測されやすい文字列を含まないこと.
```

許容されない使用例:

```markdown
- Python は教育・採用・再現性の "共通言語"
    - 教育: チュートリアルや教材, 実装例が豊富で, 学習コストが読みやすい.
    - 採用: リクルート市場で Python 経験者が見つけやすく, チームとしての生産性が上がる.
    - 再現性: 実験コードが共有されやすく, 再現性・レビューもしやすい.
```

### 記号とインデント

- 箇条書きの記号はハイフン `-` のみを使用します. アスタリスク `*` は使用してはいけません.
- ネストは半角スペース 4 つでインデントします.

```markdown
- AAA
- BBB
- CCC
    - XXX
        - ααα
    - YYY
        - βββ
    - ZZZ
        - γγγ
```
