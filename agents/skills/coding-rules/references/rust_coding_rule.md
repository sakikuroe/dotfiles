# Rust コーディングルール

## このドキュメントの目的

- 本ドキュメントは, このリポジトリで Rust のコードを書くときの基本ルールをまとめます.
- 新しくコードやモジュールを追加する場合や, 既存コードを修正する場合は, ここに書かれた方針に従ってください.

## コーディングスタイル

- インデントには半角スペース 4 個分を用います.
- 各コード・ブロックは原則として 80 文字以内とし, 長くなる場合は適宜折り返します.
- コードの整形には `rustfmt` の標準設定を用います. 特別な理由がある場合のみ設定を変更し, その理由をコメントやドキュメントに残してください.

## 型・変数宣言

- 型アノテーションは, `const` および `static` を除いて, 左辺で型を明示しないでください.
- 通常は, イテレーターや `parse` など, 型推論が必要な場合に右辺での型注記 (ターボフィッシュなど) を用います.
- やむを得ず型推論ができない場面を除き, 左辺に型を書くことは避けてください.

例:

```rust
let a = 0_u32;
let v = v.iter().collect::<Vec<usize>>();
let value = "42".parse::<u64>().unwrap();
```

`const` と `static` の例:

```rust
const N: usize = 100;
static FLAG: AtomicBool = AtomicBool::new(false);
```

- type による type alias 宣言 (`type Alias = ...;`) は禁止します.

## use の方法

- `use` 文で関数や型のインポートを行う場合, 関数や型そのものを直接インポートせず, 原則として親モジュールまでのパスを `use` で指定してください. 関数や型を `use` で直接 (例: `use super::front_of_house::hosting::add_to_waitlist;`) インポートし, 本体から `add_to_waitlist();` のように呼び出すことは禁止します. このルールにより, 関数や型の定義位置が明確となり, ローカル定義か否かの判別性が向上します.

- 例 1:

```rust
use super::front_of_house::hosting;

fn main() {
    hosting::add_to_waitlist();
}
```

- 例 2:

```rust
use rand::distr::{self, Distribution};

fn main() {
    let mut rng = rand::rng();
    let dist = distr::Uniform::new(0.0_f32, 1.0_f32).unwrap();
    let x = dist.sample(&mut rng);
    println!("{x}");
}

```

- ローカルの module に関しては, `crate` を使用せず, `super` を使用してパスまでたどってください. すなわち, 絶対パスを使わず, 相対パスを使用します.
- `use` 文における `as` による別名付け (例: `use foo as bar;`) は禁止します.
- `use` が無くても書ける場合であっても, 本体のコードでは `std::...` のような crate root から始まる完全修飾パスを原則として使用しません. 必ず `use` で "1 つ上のモジュール" まで取り込み, `module::Item` の形で参照してください.

    - OK:

    ```rust
    use std::error;

    impl error::Error for FatalError {}
    ```

    - NG:

    ```rust
    impl std::error::Error for FatalError {}
    ```

- 上記の "完全修飾パスを避ける" ルールは, `impl` ヘッダーだけでなく, trait bound, 型注釈, 関数の戻り値型, `where` 句など, 型が現れるあらゆる位置に適用します.
- 名前の衝突を避けたい場合は, `as` での別名付けではなく, 取り込む単位をモジュールに揃えた上で, `io::Error` と `error::Error` のようにモジュール経由で区別します.

    ```rust
    use std::{error, io};

    fn f() -> io::Result<()> {
        let _ = error::Error::source;
        Ok(())
    }
    ```

- 同じルートの `use` が複数並ぶ場合は, 可能な限り波括弧でまとめます. これにより import の見通しが良くなり, 重複したパス記述を減らすことができます.

    - OK:

    ```rust
    use std::{error, fmt, io, mem, process};
    ```

    - NG:

    ```rust
    use std::error;
    use std::fmt;
    use std::io;
    ```

    - 親モジュールと子モジュールの両方が必要な場合は, `self` を使ってまとめる.

    ```rust
    use std::io::{self, prelude};
    ```

## コメントおよびドキュメント

### 通常コメント (`//`)

- 言語と文体:
    - コメントは全て日本語で記述します.
    - コードの逐語訳ではなく, コードが表現する意図や背景にあるロジックを説明する文章になるように記述してください.
- 配置と粒度:
    - 原則として, 変数の宣言・初期化, ループの開始, 条件分岐, 主要な計算ステップごとにコメントを付与します.
    - "コードの塊 (ブロック)" 単位ではなく, "論理的な 1 ステップ" 単位で細かく記述します. コードを読まずにコメントだけを読んでも, 処理の流れとアルゴリズムが理解できる粒度を目指します.
- 記述内容:
    - なぜその初期値なのか, なぜその条件でループを抜けるのか, なぜその計算式なのか (数学的根拠や最適化の意図) を説明します.
        - 例: "偶数は素数になり得ないため, 3 から開始し, 2 ずつ増加させる."
    - その行で期待している状態や, 守られるべき不変条件を説明します.
        - 例: "二乗計算でのオーバーフローを防ぐため, u128 にキャストして比較する."

### ドキュメントコメント (`///`, `//!`)

- ドキュメントコメントは日本語で記述します.
- クレート全体の説明は, ファイルの最上部に `//!` で記述し, クレートの概要 (機能, 特徴など) を記載します.
- あらゆるアイテム (`trait`, `struct`, `enum`, `fn`, `mod` など, pub なものもそうでない non pub なもの両方とも) に, 直前に `///` で詳細なドキュメントコメントを付与します.
- 関数のドキュメントコメントには, 次のようなセクションを設けます.
    - 概要.
    - `# Args`: 引数の一覧と意味. 引数や内部状態に関する制約があればその旨も明示します.
    - `# Returns`: 戻り値の意味.
    - `# Examples`: 実行可能な Rust コードブロックによる使用例.
- `Args` および `Returns` では, 型だけでなく, その値が何を表し, どのように使われるかも説明します.

### エラーの書式

- エラーメッセージは英語で記述します.
- `assert_eq!` マクロでは, 第 1 引数に期待値, 第 2 引数に実際の値を渡します. すなわち, (期待値, 実際の値) の順にします.


## ルールを満たしたコード例

以下に, 全てのルールを満たしたコード例を示します.

コメントの量や文体, 語尾などについて, 以下のコードを参考に,
なるべく近づけるようにコーディングしてください.

```rust
//! 素数生成に関連する機能を提供するモジュール.
//!
//! このモジュールは, 指定された個数の素数をシンプルかつ確実に生成するための関数を含む.
//! 古典的な "試し割り" 法を採用し, 各候補数に対して既知の素数による剰余判定を行うことで
//! 素数性を検証する.

/// 最初の `n` 個の素数を生成する.
///
/// # Args
/// - `n` - 生成する素数の個数. 生成される素数の最大値は `u64` の範囲内に収まる必要がある.
///
/// # Returns
/// 計算された素数が昇順に格納された `Vec<u64>` を返す.
///
/// # Panics
/// - メモリの割り当てに失敗した場合 (`n` が極端に大きい場合など) にパニックする.
///
/// # Complexity
/// - 時間計算量: $O(n \sqrt{p_n})$
///   - $p_n$ は $n$ 番目の素数を表す. 各素数を見つけるために, その平方根以下の素数で割り算を行う.
/// - 空間計算量: $O(n)$
///   - 見つかった素数を保持するために, $n$ に比例したメモリを使用する.
///
/// # Examples
/// ```
/// // 0 個の場合は空のベクタを返す.
/// assert_eq!(Vec::<u64>::new(), literate_primes::generate(0));
///
/// // 最初の 1 個の素数.
/// assert_eq!(vec![2], literate_primes::generate(1));
///
/// // 最初の 5 個の素数.
/// assert_eq!(vec![2, 3, 5, 7, 11], literate_primes::generate(5));
/// ```
#[must_use]
pub fn generate(n: usize) -> Vec<u64> {
    if n == 0 {
        return Vec::new();
    }

    // 素数を格納するためのベクタを事前に確保する.
    // 素数は見つかり次第, 順次このリストに追加される.
    let mut primes = Vec::with_capacity(n);

    // 最初の素数は 2.
    primes.push(2);

    // 次の素数の候補値. 偶数は素数になり得ないため, 奇数のみを対象として
    // 3 から開始し, 2 ずつ増加させる.
    let mut candidate = 3_u64;

    // 指定された個数の素数が見つかるまでループを継続する.
    while primes.len() < n {
        // 現在の候補値が素数であるかどうかを判定するためのフラグ.
        // 以下の検証ループで割り切れる素数が見つかった場合, false に更新する.
        let mut is_prime = true;

        // 候補値が合成数であるかを判定するために, 既知の素数で試し割りをする.
        // 候補値の平方根以下の素数のみを確認すれば十分.
        // なぜならば, これより大きな素数で割り切れる場合, その商は必ず平方根より小さくなり,
        // すでにチェック済みであるため.
        for &prime in &primes {
            // 素数の二乗が候補値を超えた場合, これ以上チェックする必要はない.
            // 注意: 二乗計算でのオーバーフローを防ぐため u128 にキャストして比較する.
            let p = prime as u128;
            if p * p > candidate as u128 {
                break;
            }

            // 割り切れる場合, 候補値は素数ではない.
            if candidate % prime == 0 {
                is_prime = false;
                break;
            }
        }

        // 素数であると判定された場合, リストに追加する.
        if is_prime {
            primes.push(candidate);
        }

        candidate += 2;
    }

    primes
}
