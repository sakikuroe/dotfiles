## 不具合の原因と発生した経緯

<!--
(調査後に追記する。起票時点で不明な場合は空欄でよい。)

本セクションには、なぜその不具合が発生していたのか、その背景と原因を記述する。
「どこをどう直したか」は PR で確認できるため、ここでは扱わない。
ここに記述するのは、不具合が混入した経緯や、なぜ今まで検出されなかったのかといった
文脈情報であり、将来の類似不具合の防止に役立てることを目的とする。

以下の例を参考にすること。

適切な例:

```md
  CSV レスポンスを UTF-8 で出力していたが、BOM (Byte Order Mark) を付与して
  いなかった。UTF-8 の CSV ファイルを BOM なしで開くと、Excel は OS の
  デフォルトエンコーディング (日本語 Windows では Shift_JIS) として
  解釈するため文字化けが発生していた。

  この不具合は初期リリース時から存在していたが、当時はテストデータに
  ASCII 文字のみを使用していたため検出されなかった。
```

  原因 (BOM 未付与) と、その原因が不具合を引き起こすメカニズム
  (Excel の Shift_JIS フォールバック)、そして混入した経緯
  (テストデータに日本語が含まれていなかった) が記述されている。
  一方で、修正内容 (BOM を付与した) は PR に委ね、ここには書いていない。

不適切な例:

```md
  エンコーディングの問題。
```

  何のエンコーディングがどう問題だったのかが分からず、
  なぜそうなっていたのかという経緯も書かれていない。
-->
