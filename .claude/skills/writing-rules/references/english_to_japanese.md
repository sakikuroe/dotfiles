# 英語から日本語への翻訳ルール

## 基本方針

外国語を翻訳する際は、直訳に寄せすぎず、意味を崩さない範囲で自然な日本語の文章になるようにすること。
以下の個別ルールも併せて参照し、訳文の品質を保つこと。

## 個別ルール

### 1. 語句を一対一で機械的に置き換えない

単語・表現を辞書的に一対一で置き換えず、文脈・トーン・文体レベルに応じた自然な表現を選ぶこと。

| | 原文 | 悪い例 | 良い例 |
|---|---|---|---|
| 例1 | However、we cannot rule out ..。| しかし、私たちは…除外することができません | ただし、…ことは否定できません |
| 例2 | If you notice any abnormal behavior ..。| もしあなたが異常な振る舞いに気づいたなら | 通常と異なる挙動にお気づきの際は |

### 2. 英語特有の表現には訳注を添える

原文の意味が日本語に直訳しただけでは伝わりにくい慣用表現・比喩・文化的背景を持つ語句には,
【注：…】の形で補足を挿入すること。

| | 原文 | 悪い例 | 良い例 |
|---|---|---|---|
| 例1 | The project was a moonshot effort from the start。| そのプロジェクトは最初からムーンショットな取り組みでした | そのプロジェクトは最初から非常に野心的な【注：原文 "moonshot"。成功の保証はないが、実現すれば革命的な影響をもたらす野心的な挑戦を指す】取り組みでした |
| 例2 | We need to move the needle on user engagement。| ユーザーエンゲージメントのニードルを動かす必要があります | ユーザーの関与度を実質的に改善する【注：原文 "move the needle"。計器の針が動くほど顕著な変化をもたらす、という意味の慣用表現】必要があります |

### 3. 文の分割・統合を柔軟に行う

原文の文数や文の切れ目にとらわれず、日本語として読みやすくなるなら文を分割・統合・語順の変更を行うこと。原文の構造を忠実に再現することより、伝えたい内容が正確かつ自然に伝わることを優先すること。

| | 原文 | 悪い例 | 良い例 |
|---|---|---|---|
| 例1 | The system detects the error。It then notifies the administrator。| システムはエラーを検知します。その後、管理者に通知します。 | システムがエラーを検知すると、自動的に管理者へ通知されます。 |
| 例2 | Before deploying、ensure that all tests have passed、all dependencies are up to date、and the staging environment has been validated by the QA team。| デプロイする前に、すべてのテストが通過していること、すべての依存関係が最新であること、そしてステージング環境がQAチームによって検証されていることを確認してください。 | デプロイ前に以下を確認してください。テストがすべて通過していること、依存関係がすべて最新であること、QAチームによるステージング環境の検証が完了していること。 |
