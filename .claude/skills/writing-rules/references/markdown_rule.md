# Markdown 機能ルール

Markdown の記法を使用する場合は、以下のルールに従うこと。

> 本ドキュメントでは、禁止記法を説明するためにコードブロック内にそれらが現れることがあるが、あくまで例示が目的であり、本文の記法としては使用しない。

## 使用できる記法

以下の記法のみ使用すること。

- 見出し (`#` ~ `######`)
- 数値なしの箇条書き
- 数値付きの箇条書き (順序や手順が意味を持つ場合)
- リンク (`[表示テキスト](https://...)`)
- 打ち消し線
- インラインコード / コードブロック
- 引用
- 数式

太字・斜体など、上記にない記法は一切使用しないこと。

## 見出し

見出しは、そのあとに続く内容を簡潔に表すためだけに使用すること。したがって、階層構造をもたせる目的だけで使用したり、意味のない長い表題を付けたりしてはならない。また、見出しの先頭に `1.` や `1.1` のような番号を付けないこと。

正しい使用例:

```markdown
## VS Code を使用するメリット

VS Code というエディターを紹介する。

### 充実した基本機能 (検索・置換)

標準搭載の検索・置換機能が強力で、正規表現やファイル横断検索、除外設定などを直感的に扱うことができる。
大規模なプロジェクトでも目的の箇所を素早く見つけられ、変更も一括で安全に反映しやすいため、作業の効率と品質が向上する。

### Git との連携

VS Code には Git との連携機能が標準で備わっており、コミットやブランチの切り替え、差分の確認などをエディター上で直感的に行うことができる。ターミナルでコマンドを打たなくても変更内容を視覚的に把握できるため、Git の操作に不慣れな人でも扱いやすい。また、GitHub 連携の拡張機能を導入すれば、プルリクエストの作成やレビューもエディター内で完結できる。
```

## 箇条書き

箇条書きは、前後の文脈とあわせて内容を見やすく整理するために使用するものであり、単に内容を列挙するためだけに使用しないこと。序や手順が意味を持つ場合は数値付きの箇条書きを、それ以外の場合はハイフンの箇条書きを使うこと。それ以外の作業手順は原則として散文で記述する ([prose_structure.md](./prose_structure.md) を参照)。

正しい使用例 1:

```markdown
アルゴリズムについて詳しくなることで、

- 問題解決力の向上: 複雑な課題を分解し、難しい問題を解けるようになる。
- 論理的思考力の向上: 内部で行われる処理を予測し、直感に頼らない一貫性のある判断や説明ができるようになる。
- コストとリソースの最適化: 限られた資源のなかで、高速に動くプログラムを実装できるようになる。

といったメリットがある。
```

正しい使用例 2:

```markdown
パスワードポリシーとして、以下の条件を満たす必要がある。

- 長さ: 12 文字以上であること。
- 文字種: 英大文字、英小文字、数字、記号をそれぞれ 1 つ以上含むこと。
- 禁止事項: ユーザー名や誕生日など推測されやすい文字列を含まないこと。
```

許容されない使用例:

```markdown
- Python は教育・採用・再現性の「共通言語」
    - 教育: チュートリアルや教材、実装例が豊富で、学習コストを見積もりやすい。
    - 採用: 採用市場で Python の経験者を見つけやすく、チームとしての生産性が上がる。
    - 再現性: 実験コードを共有しやすく、再現やレビューも行いやすい。
```

### 記号とインデント

箇条書きの記号にはハイフン `-` のみを使用し、アスタリスク `*` は使用しないこと。また、ネストする場合は半角スペース 4 つでインデントすること。

```markdown
- AAA
- BBB
- CCC
    - XXX
        - ααα
    - YYY
        - βββ
    - ZZZ
        - γγγ
```
